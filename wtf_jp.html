透過的な暗号化メッセージング

一体何ですか？

これは、大部分のメッセージングアプリケーション（WhatsApp、ハングアウトなど）を実行している大量のデータ、または「ボンネットの下で」何が起こっているのかについて透明性がないことが限られているコンパイル済みアプリに対応しています。 外国政府の潜在的な政治動機（テレグラムを見て）はもちろんです。

強力な暗号（256ビットAES）を使用してショートメッセージを暗号化し、携帯電話のテキストメッセージ（160文字）内で動作させる。 このウェブサイトが利用できないときに使用するコンピュータまたはスマートフォンに保存します。

インストールするアプリはありません。 サインアップするサービスはありません。 すべてあなたのブラウザに保存されています。 サーバに格納されたり、送信されたりすることはありません。 100％Javascript。 すべてのコードは未検討でレビュー可能 - ここに

なぜあなたはこれを作ったのですか？

多くの「安全なメッセージング」アプリがあります。 問題は、実際に実行しているものを完全に監査できないことです。 彼らはコンパイルされ、アプリストアやGoogle Playに移動したりコンパイルされたりすると、アプリの分析などが若干変更されることがあります。

covert.chatは、見やすく、監査やレビューが容易なSMSとJavaScriptを使用します。

covert.chatはjQueryライブラリを使用していません（これは最も優れていますが、セキュリティの見直しには恐ろしく苦しいです）。

covert.chatはあらゆる種類の追跡や悪化を引き起こす可能性のあるGoogle、Facebook、Twitter、Instagram、Foursquare、MySpaceなどのログインや分析JavaScriptを使用していません。

それは何ですか

  1. IV =タイミング（マウスの動き、スマートフォン/タブレットの傾き）から収集された12のランダムなバイトに加えて、現在の時間（チャットタイムラインのシーケンスに使用され、セキュリティ上の問題ではない）には4バイト。
  2. K = SHA256_HMAC(IV, パスフレーズ)
  3. メッセージ文字列をバイト配列に変換します。
  4. CTRモードで256ビットAES暗号化を使用して、パディングされたメッセージMを暗号文Cに暗号化します。: C = AES-256-CTR(K,IV,M)
  5. base64を使って暗号化されたCで、テキストメッセージを＃-link（サーバに送信されないもの）として送信します。

復号化は基本的に逆です。ユーザーがクリックするURLは、サーバーにデータを送信する？-link形式を使用しません。 ＃-link形式を使用して、暗号化された暗号文をブラウザにのみ保持します。

How to use it?

You can just go to https://covert.chat/#l=jp or if you are a little more paranoid you can create a small HTML file and rely on HTML's Subresource Integrity to load the specific version of the code you want.

https://covert.chat/に行くこともできますし、もう少し妄想的であれば、小さなHTMLファイルを作成し、HTMLのSubresource Integrityを利用して特定のバージョンのコードを読み込むこともできます。

<html> <script src="https://s3.ca-central-1.amazonaws.com/cdn.covert.chat/2018-04-09-gz.js" integrity="sha256-h40Z2gQ4eHr6kLqiPPZRcj0Jv+OLlsivDuXEI0unFnE=" crossorigin="anonymous"></script> </html>

計画されている機能

  • アドレス帳（JavaScript localStorageに保存されている）の自動完成
  • One-Time Diffie-Hellman鍵交換を使用して非常に強力なパスワードを選択する（およびlocalStorageに保存する）
