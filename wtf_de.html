Transparente verschlüsselte Nachrichten

Was zur Hölle ist das?

Dies ist eine Reaktion auf Big-Data, auf der die meisten Messaging-Apps (WhatsApp, Hangouts usw.) oder kompilierte Apps mit eingeschränkter Transparenz laufen, was "unter der Haube" passiert. Ganz zu schweigen von möglichen politischen Motiven ausländischer Regierungen (mit Blick auf Ihr Telegramm).

Verschlüsseln Sie kurze Nachrichten mit starker Kryptographie (256bit AES) und lassen Sie sie in einer Mobiltelefon-Textnachricht (160 Zeichen) arbeiten. Speichern Sie auf Ihrem Computer oder Smartphone, um zu verwenden, wenn diese Website nicht verfügbar ist.

Keine zu installierende App Kein Dienst für die Anmeldung. Alles in Ihrem Browser gespeichert. Nichts wird gespeichert oder über einen Server übertragen. 100% Javascript. Der gesamte Code ist nicht verkleinert und steht zur Überprüfung bereit hier.

Warum hast du das gemacht?

Es gibt viele "Secure Messaging" -Apps. Das Problem ist, dass Sie nicht vollständig prüfen können, was tatsächlich ausgeführt wird, wenn Sie sie verwenden. Sie werden kompiliert und oft leicht verändert (App-Analysen usw.), wenn sie im App Store oder Google Play spielen oder kompiliert werden.

covert.chat verwendet SMS und JavaScript, die nicht verschleiert und einfach zu auditieren und zu überprüfen sind.

covert.chat verwendet nicht die jQuery-Bibliothek (die am ausgezeichnetsten ist, aber für Sicherheitsfragen schrecklich schmerzhaft ist).

covert.chat verwendet keine Google-, Facebook-, Twitter-, Instagram-, Foursquare-, Myspace-, usw. Logins oder Analytics-JavaScript, die alle Arten von Tracking oder schlechter injizieren können.

Was es macht

  1. IV = 4 Bytes für die aktuelle Zeit (wird verwendet, um Chat-Timelines abzuspeisen, kein Sicherheitsproblem) plus 12 zufällige Bytes, die aus Timings, Mausbewegungen, Smartphone- / Tablet-Kippen gesammelt wurden.
  2. K = SHA256_HMAC(IV, passphrase)
  3. Konvertiert die Nachrichtenzeichenfolge in ein Array von Bytes.
  4. Verschlüsselt die aufgefüllte Nachricht M im CTR-Modus mit 256-Bit-AES-Verschlüsselung in Chiffretext C: C = AES-256-CTR(K,IV,M)
  5. Sendet eine Textnachricht als einen # -Zweig (nichts, der an den Server gesendet wird) mit dem verschlüsselten Text C, der mit Base64 codiert ist

Entschlüsselung ist grundsätzlich umgekehrt - und die URL, auf die Benutzer klicken, verwendet nicht das? -Link-Format, das Daten an Server sendet. Es verwendet das # -link-Format, das den verschlüsselten Chiffretext nur in Ihrem Browser speichert.

Wie man es benutzt?

Sie können einfach zu https://covert.chat/#l=de gehen oder, wenn Sie etwas paranoid sind, können Sie eine kleine HTML-Datei erstellen und sich auf die Subresource Integrity von HTML verlassen, um die spezifische Version des gewünschten Codes zu laden.

<html> <script src="https://s3.ca-central-1.amazonaws.com/cdn.covert.chat/2018-04-09-gz.js" integrity="sha256-h40Z2gQ4eHr6kLqiPPZRcj0Jv+OLlsivDuXEI0unFnE=" crossorigin="anonymous"></script> </html>

Geplante Funktionen

  • Adressbuch (in JavaScript localStorage gespeichert) wird automatisch vervollständigt
  • Einmaliger Diffie-Hellman-Schlüsselaustausch, um ein sehr starkes Passwort zu wählen (und speichern Sie es in localStorage)
