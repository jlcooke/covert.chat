투명한 암호화 메시징

What the heck is this?

이는 메시징 앱 (WhatsApp, Hangouts 등)의 대부분을 실행하는 대용량 데이터 또는 "내부적 인 현상"에 대해 투명성이 제한된 컴파일 된 앱에 대한 대응입니다.

강력한 암호화 (256 비트 AES)를 사용하여 짧은 메시지를 암호화하고 휴대 전화 문자 메시지 (160 문자) 내에서 작동하게하십시오. 이 웹 사이트를 사용할 수 없을 때 사용할 컴퓨터 또는 스마트 폰에 저장하십시오.

설치할 앱이 없습니다. 가입 할 서비스가 없습니다. 모든 것이 브라우저에 저장됩니다. 서버를 통해 저장되거나 전송되지 않습니다. 100 % Javascript. 모든 코드는 축소되지 않았으며 여기 에서 검토 할 준비가되었습니다. 너와 너 자신과 자물쇠 이리.

왜 이걸 만들었 니?

모든 종류의 추적 또는 악화를 유발할 수있는 Google, Facebook, Twitter, Instagram, FourSquare, MySpace 등의 로그인 또는 분석 JavaScript는 사용하지 않습니다.

covert.chat 잘 보이지 않고 감사 및 검토가 쉬운 SMS 및 JavaScript를 사용합니다.

covert.chat jQuery 라이브러리를 사용하지 않는다. (가장 뛰어나지 만, 보안을 검토하기에는 끔찍한 일이다.)

covert.chat는 Google, Facebook, Twitter, Instagram, FourSquare, MySpace 등의 로그인이나 모든 종류의 추적 또는 악화를 유발할 수있는 분석 JavaScript를 사용하지 않습니다.

그것이하는 일

  1. IV = 타이밍, 마우스 움직임, 스마트 폰 / 태블릿 기울이기에서 수집 된 16 개의 임의 바이트
  2. K = SHA256_HMAC(IV, 암호문)
  3. 메시지 문자열을 바이트 배열로 변환합니다.
  4. CTR 모드에서 256 비트 AES 암호화를 사용하여 패딩 된 메시지 M을 암호문 C로 암호화합니다.: C = AES-256-CTR(K,IV,M)
  5. base64를 사용하여 암호문 C로 인코딩 된 텍스트 메시지를 # - 링크 (서버에 전송되지 않음)로 보냅니다.

암호 해독은 기본적으로 반대이며 사용자가 클릭하는 URL은 서버에 데이터를 보내는? 링크 형식을 사용하지 않습니다. # -link 형식을 사용하여 암호화 된 암호문을 브라우저에만 유지합니다.

사용 방법?

https://covert.chat/#l=kr로 가거나 작은 편집증 환자라면 작은 HTML 파일을 만들고 HTML의 Subresource Integrity를 사용하여 원하는 특정 코드 버전을로드 할 수 있습니다.
