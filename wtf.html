Transparent Encrypted Messaging

What the heck is this?

This is in response to big-data running most of the messaging apps (WhatsApp, Hangouts, etc) or compiled apps with limited to no transparency on what's going on "under the hood". Not to mention potential political motives of foreign governments (looking at you Telegram).

Encrypt short messages using strong cryptography (256bit AES) and have it work inside a mobile phone text message (160 characters). Save on your computer or smartphone to use when this website is unavailable.

No app to install. No service to sign up to. Everything stored in your browser. Nothing stored or even transmitted through a server. 100% Javascript. All code is un-minified and ready for review here.

Why did you make this?

There are lots of "secure messaging" apps out there. Problem is - you can't fully audit what is actually running when you use them. They're compiled, and often slightly altered (app analytics, etc) when they go on to the app store or google play or whatever.

covert.chat uses SMS and JavaScript that is un-obfuscated and easy to audit and review.

covert.chat doesn't use the jQuery library (which is most excellent, but horrendously painful to review for security).

covert.chat doesn't use any Google, Facebook, Twitter, Instagram, FourSquare, MySpace, etc logins or analytics JavaScript which may inject all kinds of tracking or worse.

What it does

  1. IV = 4 bytes for the current time (used to sequence chat timelines, not a security issue at all) plus 12 random bytes collected from timings, mouse movements, smartphone/tablet tilting.
  2. K = SHA256_HMAC(IV, passphrase)
  3. Converts message string into an array of bytes.
  4. Encrypts the padded message M into ciphertext C using 256-bit AES encryption in CTR mode: C = AES-256-CTR(K,IV,M)
  5. Sends text message as a #-link (nothing sent to the server) with the ciphertext C encoded using Base64

Decryption is basically the reverse - and the URL users click on do not use the ?-link format which sends data to servers. It uses the #-link format which keeps the encrypted ciphertext on your browser only.

How to use it?

You can just go to https://covert.chat/ or if you are a little more paranoid you can create a small HTML file and rely on HTML's Subresource Integrity to load the specific version of the code you want.

<html> <script src="https://s3.ca-central-1.amazonaws.com/cdn.covert.chat/2018-2018-04-09-gz.js" integrity="sha256-h40Z2gQ4eHr6kLqiPPZRcj0Jv+OLlsivDuXEI0unFnE=" crossorigin="anonymous"></script> </html>
